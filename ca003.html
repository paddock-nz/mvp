Nitrogen product database

Build a Python library that takes a fertiliser product name and weight applied and returns kilograms of elemental nitrogen. This is your first deliverable on the Paddock project. Read every tab before writing any code.

Definition of done: A function called calculate_nitrogen_kg() takes a product name (string) and weight applied (float, kg/ha or total kg), fuzzy-matches the product to the database, and returns elemental nitrogen in kg. It raises a clear error if no match is found above threshold. The Architect can import this library from the rule engine with zero changes.

What you are building

The complete picture

The Compliance Agent needs to know, for any nitrogen fertiliser a NZ dairy farmer has applied, exactly how many kilograms of elemental nitrogen that represents. Your job is to build the Python library that does this calculation reliably.

This library will be used inside the rule engine (built in Sprint 2 by the Architect). It is a dependency of almost everything else in the product. If the N% figures are wrong, every compliance calculation in the system will be wrong. Take this seriously.

Why this is harder than it sounds

A farmer types "urea" — easy. But farmers also type "Urea 46", "UREA", "urea fert", "sustain", "SustaiN 46", "sustain ammo 36n", "can", "calcium ammonium nitrate", or just "the white stuff". Your library needs to handle all of these gracefully, match to the right product, and calculate correctly.

You will build a fuzzy match function — a function that finds the closest match in your database even when the spelling isn't exact. This is a real-world AI/data technique used in production systems everywhere.

Architect's rules you must follow

Rule 1 — n_pct is always a float

Never store N% as an integer. 46 is wrong. 46.0 is correct. This prevents subtle bugs when the rule engine does arithmetic.

Rule 2 — weight_kg is total kg, not kg/ha

Your function receives total kilograms already applied to the paddock. The caller (rule engine) handles the area multiplication. Never do area calculations inside this library.

Rule 3 — never silently return a wrong match

If your fuzzy match score is below match_threshold, raise ProductNotFoundError. Do not return a guess. A wrong N% calculation is worse than an error — it produces silently incorrect compliance data.

Rule 4 — the library must be side-effect free

Importing your library must not print anything, connect to any API, or write any files. It only loads products.json once at import time. Everything else is pure functions.

Rule 5 — supplier_code must be unique and stable

Once you assign a supplier_code, never change it. The rule engine stores it in the database to link back to your product. Changing a code after Sprint 2 will break existing records.

Step 1 — load the database at import time

The first thing your module does when it's imported is load products.json into memory. We do this once at the top, not inside every function — loading a file is slow and you don't want to do it 1000 times.

n_converter.pystart here
"""
Paddock N% Conversion Library
Converts NZ fertiliser product names to elemental nitrogen kg.
CA-003 | Sprint 1 | Junior Dev
"""
import json
import os
from rapidfuzz import fuzz, process

# ── Load product database at import time ──────────────────────────

# __file__ is the path to THIS file (n_converter.py)
# os.path.dirname gets the folder it's in
# os.path.join builds the path to products.json in the same folder
_DB_PATH = os.path.join(os.path.dirname(__file__), "products.json")

with open(_DB_PATH, "r", encoding="utf-8") as _f:
    _DB = json.load(_f)["products"]

# Pre-build a flat list of all searchable names:
# canonical names + all aliases, mapped back to the product
_SEARCH_TERMS: list[tuple[str, dict]] = []
for product in _DB:
    # Add the canonical name
    _SEARCH_TERMS.append((product["canonical_name"].lower(), product))
    # Add all aliases
    for alias in product.get("aliases", []):
        _SEARCH_TERMS.append((alias.lower(), product))
// new concept: module-level code Code that is outside any function runs exactly once when Python imports the module. By loading the JSON here (not inside a function), we load it once when the program starts and reuse it for every function call. This is a common performance pattern in Python.

Step 2 — the custom exception

Add this right after the database loading code. Custom exceptions let the caller handle errors specifically — the rule engine can catch ProductNotFoundError and log it differently from a generic error.

class ProductNotFoundError(Exception):
    """Raised when no fertiliser product matches the query above threshold."""
    def __init__(self, query: str, best_match: str = None, best_score: int = 0):
        self.query = query
        self.best_match = best_match
        self.best_score = best_score
        msg = f"No product found for '{query}'"
        if best_match:
            msg += f". Closest match: '{best_match}' (score: {best_score})"
        super().__init__(msg)

Step 3 — the fuzzy match helper

This is the core of the library. rapidfuzz compares your query against every name in the database and returns a similarity score from 0 to 100. We use WRatio which handles different word orders and partial matches — ideal for product names.

n_converter.pyinternal helper — note the leading underscore
def _find_best_match(
    query: str,
    threshold: int = 75
) -> tuple[dict | None, int]:
    """
    Internal function. Find the best matching product for a query string.

    Returns a tuple: (product_dict or None, score)
    The leading underscore signals to other developers: don't call this
    from outside the module. Use calculate_nitrogen_kg() instead.
    """
    # Normalise the query: lowercase, strip whitespace
    query_clean = query.lower().strip()

    # Get all the searchable terms (just the strings, not the product dicts)
    terms = [term for term, _ in _SEARCH_TERMS]

    # Ask rapidfuzz for the best match
    # process.extractOne returns: (matched_string, score, index)
    result = process.extractOne(
        query_clean,
        terms,
        scorer=fuzz.WRatio,   # WRatio handles word order variations
        score_cutoff=0,          # don't cut off — we want to see the best even if low
    )

    if result is None:
        return None, 0

    matched_term, score, index = result

    # Get the product dict that corresponds to the matched term
    product = _SEARCH_TERMS[index][1]

    if score >= threshold:
        return product, int(score)
    else:
        # Return the product for error reporting, but flag as below threshold
        return None, int(score)

Step 4 — calculate_nitrogen_kg (the main function)

n_converter.pymain public function
def calculate_nitrogen_kg(
    product_name: str,
    weight_kg: float,
    match_threshold: int = 75
) -> dict:
    """
    Calculate elemental nitrogen from fertiliser product name and weight.
    See Architect Input tab for full docstring.
    """
    # Validate inputs first — fail fast with clear errors
    if not isinstance(product_name, str) or not product_name.strip():
        raise ValueError("product_name must be a non-empty string")

    if weight_kg <= 0:
        raise ValueError(f"weight_kg must be positive, got: {weight_kg}")

    # Find the best match
    # _find_best_match returns (product_or_None, score)
    product, score = _find_best_match(product_name, threshold=match_threshold)

    if product is None:
        # Get the score of what was closest, even if below threshold
        _, best_score_anyway = _find_best_match(product_name, threshold=0)
        raise ProductNotFoundError(
            query=product_name,
            best_match="unknown",
            best_score=best_score_anyway
        )

    # Calculate elemental nitrogen
    # n_pct is stored as e.g. 46.0, meaning 46%
    # so we divide by 100 to get the fraction
    n_kg = round(weight_kg * (product["n_pct"] / 100), 4)

    return {
        "product_matched": product["canonical_name"],
        "n_pct":          product["n_pct"],
        "weight_kg":       round(weight_kg, 4),
        "n_kg":            n_kg,
        "match_score":     score,
        "source":          product["source"],
        "supplier_code":   product["supplier_code"],
    }

Step 5 — search_products and get_product_by_code

n_converter.pyremaining public functions
def search_products(
    query: str,
    limit: int = 5,
    threshold: int = 60
) -> list[dict]:
    """Return ranked list of matching products for UI autocomplete."""
    if not query or not query.strip():
        return []

    query_clean = query.lower().strip()
    terms = [term for term, _ in _SEARCH_TERMS]

    # extractMany returns up to `limit` matches
    results = process.extract(
        query_clean,
        terms,
        scorer=fuzz.WRatio,
        limit=limit * 3,   # get more then deduplicate
        score_cutoff=threshold
    )

    # Deduplicate by supplier_code (same product may match via name AND alias)
    seen_codes = set()
    output = []
    for matched_term, score, index in results:
        product = _SEARCH_TERMS[index][1]
        code = product["supplier_code"]
        if code not in seen_codes:
            seen_codes.add(code)
            output.append({
                "product_name":  product["canonical_name"],
                "n_pct":         product["n_pct"],
                "source":        product["source"],
                "supplier_code": code,
                "match_score":   int(score),
            })
        if len(output) >= limit:
            break

    return output


def get_product_by_code(supplier_code: str) -> dict | None:
    """Look up a product by its exact supplier code."""
    for product in _DB:
        if product["supplier_code"] == supplier_code:
            return product
    return None

# ── SECTION 4: N kg calculation accuracy ─────────────────────────

class TestNitrogenCalculation:
    def test_urea_1000kg(self):
        # 1000 kg × 46% = 460 kg N
        result = calculate_nitrogen_kg("urea", 1000)
        assert result["n_kg"] == 460.0

    def test_urea_paddock_scenario(self):
        # 150 kg/ha × 14.2 ha = 2130 kg product → 979.8 kg N
        result = calculate_nitrogen_kg("urea", 150 * 14.2)
        assert result["n_kg"] == pytest.approx(979.8, rel=1e-3)

    def test_can_500kg(self):
        # 500 kg × 27% = 135 kg N
        result = calculate_nitrogen_kg("CAN", 500)
        assert result["n_kg"] == 135.0

    def test_result_contains_all_required_keys(self):
        result = calculate_nitrogen_kg("urea", 100)
        required_keys = [
            "product_matched", "n_pct", "weight_kg",
            "n_kg", "match_score", "source", "supplier_code"
        ]
        for key in required_keys:
            assert key in result, f"Missing key: {key}"

    def test_n_pct_is_float_not_int(self):
        result = calculate_nitrogen_kg("urea", 100)
        assert isinstance(result["n_pct"], float)

        with pytest.raises(ValueError):
            calculate_nitrogen_kg("urea", -100)

    def test_empty_string_raises_error(self):
        with pytest.raises(ValueError):
            calculate_nitrogen_kg("", 100)

    def test_error_has_query_attribute(self):
        with pytest.raises(ProductNotFoundError) as exc_info:
            calculate_nitrogen_kg("not_a_product", 100)
        assert exc_info.value.query == "not_a_product"
